1. Abstract

Colr este o platforma pentru colectionarii de capace, dopuri de pluta si etichete ale diverselor bauturi alcoolice si non-alcoolice. de obiecte.

2. Introduction

Colr este mediul perfect pentru colectionari. Prin Colr, utilizatorii logati pot sa adauge obiecte pe platforma, sa isi creeze, actualizeze si vizualizeze propria lor colectie. De asemenea, se pot vizualiza si statistici referitoare la obiecte si utilizatori, topuri si se pot exporta datele expuse pe site in diferite formate (CSV, RSS, DocBook).

3. Structure

In cadrul acestui capitol, vom prezenta structura proiectului, constand in:

  • Structura bazei de date
  • Prezentarea elementelor speciale de back-end si front-end
  • Structura si functionalitatea paginilor

Astfel, in descrierea aferenta fiecarei pagini, se va respecta urmatoarea structura a ideilor:

  1. Prezentarea generala a paginii
  2. Descrierea functionalitatilor generale, cu:
    • Elemente de front-end
    • Elemente de back-end
    • Relatia cu baza de date
  3. Conditii de acces si alte elemente specifice paginii

3.1 Database

Folosim serverul si baza de date prin XAMPP. Structura acesteia este relativ simpla, cu 3 tabele:

  1. items
    • stocheaza toate colectionabilele
  2. users
    • stocheaza toate datele utilizatorilor
    • parolele stocate sunt criptate
  3. itemsowned
    • se leaga tabelele items si users, stocand toate obiectele detinute de utilizatori

3.2 Technologies

Pentru a face legatura cu baza de date, s-au folosit o serie de fisiere .php pentru API, conform paradigmei REST.

3.3 Home page

Fisierul index.html reprezinta pagina home, unde se pot gasi urmatoarele elemente:

  • Meniul, prezent pe toate paginile site-ului, este format din urmatoarele elemente:
    • Home: face redirect catre pagina de home
    • Alcoholic: afiseaza pe pagina home toate obiectele de tip alcoolic, si la actiunea de hover se afiseaza urmatorul submeniu:
      • Caps: afiseaza lista cu capacele de tip alcoolic
      • Corks: afiseaza lista cu dopurile de pluta de tip alcoolic
      • Labels: afiseaza lista cu etichetele de tip alcoolic
    • Non-alcoholic: afiseaza pe pagina home toate obiectele de tip non-alcoolic, si la actiunea de hover se afiseaza urmatorul submeniu:
      • Caps: afiseaza lista cu capacele de tip non-alcoolic
      • Corks: afiseaza lista cu dopurile de pluta de tip non-alcoolic
      • Labels: afiseaza lista cu etichetele de tip non-alcoolic
    • Username menu: contine un submeniu specific utilizatorului, cu urmatoarele elemente:
      • User panel: deschide panoul utilizator
      • My items: afiseaza lista cu elementele detinute de utilizator
      • Administrator panel: deschide panoul administrator, prezent doar in cazul in care administratorul este logat
      • Logout: permite utilizatorului sa se delogheze
  • Search bar multicriterial
  • Buton pentru adaugarea unui obiect nou
  • Containerul cu elementele afisate, pe baza criteriilor introduse, impreuna cu butonul de adaugare element in colectie
  • Graficul cu statisticile referitoare la utilizatori si obiecte

3.4 Login / Register page

Fisierul logreg.html reprezinta pagina pentru inregistrarea si autentificarea utilizatorilor, unde se pot gasi urmatoarele elemente:

  • Meniul
  • Formularul de register
  • Formularul de login

Dupa inregistrarea utilizatorului, acesta va fi redirectionat automat catre login. Acesta poate opta pentru salvarea credentialelor, bifand checkbox-ul "rememberMe", sau debifand-ul pentru a goli cookie-ul si, implicit, campurile. In cazul autentificarii cu succes, utilizatorul este redirectionat catre home

3.5 User panel

Fisierul panel.html reprezinta pagina pentru panoul utilizatorului, unde se pot gasi urmatoarele elemente:

  • Meniul
  • Change password
  • Change email

Aceasta pagina permite utilizatorului logat sa isi schimbe parola si/sau emailul. In cazul in care un client neautentificat doreste sa acceseze URL-ul, acesta nu primeste acces pe baza cookie-ului.

3.6 Admin panel

Fisierul admin.html reprezinta pagina pentru panoul administrator, unde se pot gasi urmatoarele elemente:

  • Meniul
  • Confirm password
  • User table

Aceasta pagina permite administratorului sa radieze utilizatori din baza de data, insa numai pe baza confirmarii parolei. Se foloseste cookie-ul pentru a verifica faptul ca URL-ul nu este accesat de un utilizator strain.
